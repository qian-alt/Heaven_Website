# 个人内容网站设计说明

**日期：** 2026-05-24  
**状态：** 已由用户确认  
**参考方向：** [Axi's Blog](https://axi404.top/) 的内容型个人空间结构，以及 [糖糖毬](https://tantanchugasuki.cn/) 的轻柔个人气质；网站以 Axi 的清爽内容组织为主要方向，不直接复刻现有站点。

## 1. 目标与定位

该网站是一个面向公开访问的个人内容站，主要呈现站主作为学生和技术学习者的成长路径、实践项目与博客文章。

首版目标：

- 让访客快速了解站主的身份、学习方向、经历和主要项目。
- 提供适合长期更新的 Markdown 博客空间。
- 以清爽平衡的视觉风格体现适度二次元个人气质。
- 内容维护简单，不依赖后台、数据库或用户账户系统。

首版不包含：

- 评论、登录、点赞与浏览量统计。
- 在线文章管理后台。
- 复杂动态社交功能或需要服务端存储的业务。

## 2. 信息架构

主导航包含以下入口：

- `首页`
- `博客`
- `项目`
- `友链`
- `关于`
- 主题切换按钮

### 首页

首页承担个人介绍与内容入口两项职责，包含：

1. 个人头图区：头像、昵称、一句身份描述、所在地或身份标签、GitHub 与邮箱等社交入口。
2. 关于我摘要：简短说明学习方向、兴趣和当前关注的技术主题。
3. 学习/教育经历：以卡片呈现学校、专业、阶段与相关经历。
4. 精选项目：展示 2 至 4 个重点项目，包含简短说明、技术标签及链接。
5. 最新文章：展示近期博客文章并链接至完整博客列表。
6. 页脚：版权信息、社交链接；配置备案信息时展示备案入口，未配置时不保留空位。

桌面端首页采用主内容区与最新文章侧栏并列的布局；移动端按个人介绍、经历、项目、文章的顺序单列展示。

### 博客

- `/blog` 展示文章列表、标签筛选与关键词搜索。
- `/blog/[slug]` 展示单篇文章，提供目录、代码高亮、发布日期、标签与上一篇/下一篇导航。
- 文章列表默认按发布日期倒序排列，置顶文章优先。

### 项目

- `/projects` 展示全部项目。
- 首页只选取标记为精选的项目。
- 项目卡片包含名称、简介、技术栈、仓库链接与在线演示链接（如存在）。

### 友链

- `/links` 展示友站卡片列表。
- 友链卡片包含站名、头像、简短介绍与外部访问链接。
- 卡片视觉与项目卡片保持一致，确保全站统一。

### 关于

- `/about` 展示完整个人介绍、学习经历、技术兴趣和联系方式。
- 首页仅保留适于快速浏览的摘要。

## 3. 视觉语言

### 整体方向

视觉采用“清爽平衡”方向：以适合阅读的内容站观感为优先，通过头像、淡色装饰和个性文案保留二次元个人气质，而不采用占据首屏的大面积角色海报。

### 默认浅色主题

- 背景：偏冷的浅白或浅蓝灰。
- 内容卡片：白色或半透明浅色面板，细描边和轻阴影。
- 点缀色：柔和蓝灰为主，少量浅粉用于头像边框或局部渐变。
- 文本：深灰正文与柔和的次要文字，确保长文可读。

### 暗色主题

- 使用深蓝灰背景和稍浅的面板层级。
- 高亮色继续使用低饱和蓝色与极少量柔粉。
- 暗色模式保留布局与信息层级，不另行改变品牌风格。
- 首次访问跟随系统主题；访客主动切换后记住偏好。

### 组件表达

- 导航：桌面端水平展示，移动端折叠为菜单。
- 卡片：圆角、细描边、低对比阴影，不使用强烈发光效果。
- 头像区：作为首页最强个人识别点，搭配昵称、简短签名与社交按钮。
- 标签：使用低对比胶囊样式，方便识别而不过度抢眼。
- 阅读页：保持较宽松行距、清晰标题层级和舒适的代码块展示。

## 4. 内容维护模型

网站为静态内容驱动，内容更新通过 Markdown/MDX 和结构化数据文件完成。

### 博客文章

文章位于 `src/content/blog/`，每篇 Markdown 或 MDX 文件包含以下字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `title` | 是 | 文章标题 |
| `description` | 是 | 列表及 SEO 使用的摘要 |
| `publishedAt` | 是 | 发布日期 |
| `updatedAt` | 否 | 更新日期 |
| `tags` | 是 | 至少一个文章标签 |
| `draft` | 否 | 是否为草稿，默认 `false` |
| `pinned` | 否 | 是否置顶，默认 `false` |
| `cover` | 否 | 文章封面路径 |

### 个人资料

统一配置包含：

- 站点标题与昵称。
- 头像、个人签名与简介摘要。
- 所在地或身份标签。
- GitHub、邮箱等社交链接。
- SEO 默认标题与描述。

### 经历数据

每项经历包含：

- 机构或学校名称。
- 专业、角色或方向。
- 起止时间。
- 可选的简介与外部链接。

简介与外部链接仅在配置后渲染；缺省时卡片只显示主要经历信息。

### 项目数据

每个项目包含：

- 名称与简介。
- 技术栈标签。
- 仓库链接与可选演示链接。
- 是否精选。
- 可选封面图片。

演示链接与封面图片仅在配置后显示；图片缺省时项目卡片采用纯文本布局。

### 友链数据

每个友链包含：

- 站点名称。
- 地址。
- 头像或图标。
- 简短描述。

构建阶段应校验所有结构化字段，避免内容缺失造成页面异常。

## 5. 技术方案

### 技术选型

- 框架：`Astro`
- 类型支持：`TypeScript`
- 样式：`Tailwind CSS`
- 内容：Astro Content Collections 管理 Markdown/MDX
- 部署：优先适配 `Vercel` 或 `Cloudflare Pages`

选择 Astro 的原因：

- 个人博客和内容档案以静态输出为主，Astro 能生成轻量、加载快速的页面。
- 内容集合适合校验 Markdown 前置数据，并为博客、标签、RSS 等能力提供一致数据来源。
- 只在搜索、主题切换或移动端菜单等需要交互的区域加入少量客户端逻辑，减少复杂度。

### 页面及代码组织

预期结构：

```text
src/
  components/       通用导航、卡片、主题切换、搜索与文章组件
  content/blog/     Markdown 或 MDX 博客文章
  data/             个人信息、经历、项目与友链数据
  layouts/          通用页面布局与文章阅读布局
  pages/            首页、博客、项目、友链、关于和文章路由
public/             头像、图标与静态图片资源
```

### 数据流

1. 站主新增文章或编辑数据文件。
2. 构建时 Astro 内容集合校验文章元数据和结构化配置。
3. 页面按数据生成：首页读取个人资料、经历、精选项目和最新文章；各独立页面读取对应完整数据。
4. RSS 与 sitemap 在构建期生成并随部署更新。
5. 推送代码后由托管平台发布最新静态站点。

## 6. 功能边界

首版应提供：

- 响应式页面布局。
- 浅色和暗色主题切换及本地偏好记忆。
- Markdown/MDX 文章渲染。
- 博客文章目录与代码高亮。
- 文章标签筛选和前端关键词搜索。
- 项目、经历和友链数据展示。
- 基础 SEO 元信息、Open Graph 信息、RSS 与 sitemap。
- 外部链接安全打开与图片缺省占位处理。

首版明确不提供：

- 评论、身份认证、后台管理与数据库。
- 访客点赞、文章浏览统计与 GitHub 活跃统计。
- 多语言内容体系；导航不预留语言切换按钮，后续有明确需求再扩展。

## 7. 错误处理与降级

- 构建时校验文章元数据、项目和友链字段；数据不符合规则时构建失败并显示明确错误。
- 头像、项目图或友链头像加载失败时展示站点内置占位图。
- 没有文章或项目时，各列表页显示友好的空状态，而非留空或报错。
- JavaScript 不可用时，静态内容仍可阅读；搜索和主题偏好记忆等增强功能可降级。
- 外部链接使用安全属性，避免新窗口访问带来的引用风险。

## 8. 验收与验证

首版完成时应验证：

- 站点构建成功，内容集合无数据校验错误。
- 首页、博客列表、文章详情、项目、友链与关于页可访问且布局正常。
- 桌面端与移动端导航均可使用。
- 浅色/暗色切换表现正确，且偏好可被记住。
- 文章列表排序、置顶、标签筛选、关键词搜索与目录行为符合设计。
- Markdown 内容、代码块和外部链接正常呈现。
- 头像或项目图片失败时能显示占位内容。
- 页面具备必要的标题、描述、RSS 和 sitemap 输出。
- 进行基础键盘导航、颜色对比度和语义结构检查。

## 9. 已确认决策

- 站点重点是内容优先的完整个人空间，而非纯氛围首屏。
- 用户喜欢 Axi 网站的内容组织与清爽风格。
- 首版内容范围为关于我、博客、项目、经历和友链。
- 站点身份以学生 / 技术学习者为主。
- 内容以 Markdown 和配置数据文件维护。
- 默认采用清爽浅色风格，并支持暗色切换。
- 博客首版专注阅读，不包含评论或热度功能。
- 主导航加入 `友链` 入口。
